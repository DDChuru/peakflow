Screen Guide

Click on numbered badges or hover over UI elements to learn more about each feature.

1

Industry Template Indicator

Shows that this company is using the "Technology & Software" industry template with 889 pre-configured accounts. This provides industry-specific GL structure out of the box.

Technical Details
  • Service: IndustryTemplateService
  • Templates: technology_software, professional_services, retail, manufacturing, etc.
  • Benefit: Skip manual COA setup, use proven industry structure
2

Account Category Summary Cards

Quick overview showing total accounts broken down by the 5 main categories: Assets, Liabilities, Equity, Revenue, Expenses. Shows distribution at a glance.

Technical Details
  • Data: Grouped from industryService.listAccounts()
  • Grouping: By account.type field
  • State: groupedAccounts Record
3

Add Account Button

Allows admins to add custom GL accounts beyond the industry template. Useful for company-specific tracking needs.

Technical Details
  • Permission: Admin or financial_admin roles only
  • Action: Opens account creation form
  • Validation: Unique account code, valid type, proper hierarchy
4

Account Categories Sections

Accounts organized by type (Asset, Liability, Equity, Revenue, Expense) with collapsible sections for easy navigation.

Technical Details
  • Rendering: Object.entries(groupedAccounts).map()
  • Cards: One per type with header and account list
  • Icons: Type-specific icons (trending-up for assets, etc.)
5

Account Code Ranges

Displays the standard accounting code ranges for each category (1000-1999 for Assets, 2000-2999 for Liabilities, etc.). Follows industry-standard numbering.

Technical Details
  • Standard Ranges: 1xxx=Assets, 2xxx=Liabilities, 3xxx=Equity, 4xxx=Revenue, 5xxx=Expenses
  • Purpose: Logical grouping and financial statement mapping
  • Template: Industry templates use these ranges
6

Account Hierarchy / Tree View

Accounts displayed in hierarchical order, showing parent-child relationships. Enables sub-account organization under main accounts.

Technical Details
  • Structure: account.parentAccountId field for hierarchy
  • Display: Indentation or tree UI for sub-accounts
  • Example: 1500 Computer Equipment > 1510 Laptops > 1520 Accumulated Depreciation
7

Account Details (Code, Name, Type)

Each account shows its unique code (e.g., "1100"), full name (e.g., "Bank Account - Main"), and type classification. Code is monospaced for clarity.

Technical Details
  • Type: CompanyAccountRecord
  • Fields: code (string), name (string), type (AccountType enum)
  • Display: Code in monospace, name in regular font
8

Normal Balance Display

Shows whether the account normally has a debit or credit balance. Important for proper transaction recording and reporting.

Technical Details
  • Field: account.normalBalance - "debit" or "credit"
  • Rule: Assets/Expenses=Debit, Liabilities/Equity/Revenue=Credit
  • Usage: Validation for transaction posting
9

Account Inquiry (Click to See Transactions)

"View Transactions" link allows drilling down to see all journal entries and transactions affecting this specific account. Account-level general ledger.

Technical Details
  • Navigation: Links to account inquiry page or filters journal entries
  • Shows: All ledger lines for this account with running balance
  • Purpose: Account reconciliation, audit, investigation
10

Industry-Specific Accounts Highlighted

Accounts like "Cloud Infrastructure Costs", "Software Licenses", "Software Development Revenue" are specific to the Technology & Software industry template and shown with purple badges.

Technical Details
  • Field: account.isIndustrySpecific boolean flag
  • Badge: Purple "Industry-Specific" badge vs blue "System" badge
  • Purpose: Distinguishes standard vs industry-tailored accounts
