Screen Guide

Click on numbered badges or hover over UI elements to learn more about each feature.

1

Report Header (Company Name, Period)

Standard report header showing report title, company name, and the as-of date for the trial balance. Critical context for interpreting the numbers.

Technical Details
  • Company Info: From company settings in Firestore
  • Date: User-selected as-of date for report
  • Format: Matches standard financial reporting format
2

Date Selector

Allows user to select the as-of date for the trial balance. Shows account balances as of the end of that date.

Technical Details
  • Component: Date picker with common presets (Month-end, Quarter-end, Year-end)
  • Calculation: Sums all ledger entries up to and including selected date
  • Default: Current month-end or today's date
3

Account Categories Sections

Accounts organized by type: Assets, Liabilities, Equity, Revenue, Expenses. Standard accounting classification for financial statement preparation.

Technical Details
  • Grouping: Based on account.type field
  • Order: Assets ‚Üí Liabilities ‚Üí Equity ‚Üí Revenue ‚Üí Expenses
  • Purpose: Facilitates Balance Sheet and Income Statement preparation
4

Account Code Column

Unique numeric identifier for each GL account. Displayed in monospace font for easy scanning and alignment.

Technical Details
  • Field: account.code from chart of accounts
  • Format: Monospace font (font-mono) for alignment
  • Standards: 1xxx=Assets, 2xxx=Liabilities, etc.
5

Account Name Column

Descriptive name of the GL account (e.g., "Bank Account - Main", "Accounts Receivable"). User-friendly label for each account.

Technical Details
  • Field: account.name from chart of accounts
  • Clickable: Links to account inquiry / general ledger detail
  • Display: Regular font weight, left-aligned
6

Debit Column

Shows debit balance for accounts with debit normal balance (Assets, Expenses). Right-aligned and formatted as currency.

Technical Details
  • Calculation: Sum of all debit entries - sum of all credit entries (if positive)
  • Format: formatCurrency() with ZAR locale
  • Display: Shows "-" if account has credit balance
7

Credit Column

Shows credit balance for accounts with credit normal balance (Liabilities, Equity, Revenue). Right-aligned and formatted as currency.

Technical Details
  • Calculation: Sum of all credit entries - sum of all debit entries (if positive)
  • Format: formatCurrency() with ZAR locale
  • Display: Shows "-" if account has debit balance
8

Export/Print Buttons

Export trial balance to PDF, Excel, or send to printer. PDF for distribution, Excel for further analysis, Print for hard copy.

Technical Details
  • PDF: Using pdfService for formatted report
  • Excel: XLSX export with formulas for totals
  • Print: Browser print dialog with @media print styles
9

Drill-Down to Account Detail

Clicking on an account name navigates to the general ledger detail for that account, showing all transactions with running balance.

Technical Details
  • Navigation: Routes to account inquiry page with account filter
  • Shows: All journal entries affecting this account
  • Purpose: Account reconciliation, investigation, audit trail
10

Total Debits = Total Credits Validation

Fundamental accounting equation validation. In a properly balanced system, total debits must equal total credits. Any mismatch indicates an error.

Technical Details
  • Validation: totalDebits === totalCredits
  • Error Detection: Shows warning banner if mismatch detected
  • Critical: Must be balanced before closing period or generating financial statements
