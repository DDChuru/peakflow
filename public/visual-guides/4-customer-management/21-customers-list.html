Screen Guide

Click on numbered badges or hover over UI elements to learn more about each feature.

1

Add Customer Button

Creates a new customer account with master data including contact info, credit limits, and payment terms. Opens a dialog with a comprehensive form for entering all customer details.

Technical Details
  • Component: Button with onClick handler
  • Action: Opens Create Customer Dialog with React Hook Form
  • Service: DebtorService.createDebtor()
  • Validation: Zod schema with email, credit limit validation
2

Search Bar

Real-time search filtering across customer name, email, and phone number. Includes keyboard shortcut (⌘K or Ctrl+K) for quick access.

Technical Details
  • State: searchTerm with controlled input
  • Filter Logic: Case-insensitive substring matching
  • Keyboard: Global listener for Cmd/Ctrl+K
  • Fields: Searches name, email, phone simultaneously
3

Filter by Status

Filters customers by status (Active, Inactive, Blocked) to quickly find customers needing attention or review.

Technical Details
  • Button: Opens filter dropdown menu
  • Status Options: active, inactive, blocked
  • Integration: Works alongside search filter
4

Customer Table Columns

Comprehensive customer data including Name (clickable to view details), Contact Info (email/phone), Status, Current Balance (AR), Overdue Amount, Payment Terms, and Actions menu.

Technical Details
  • Columns: Customer, Contact, Status, Balance, Overdue, Terms, Actions
  • Data Source: filteredCustomers array from state
  • Row Click: Opens view dialog with customer details
  • Styling: Sticky header, hover effects, responsive layout
5

Current Balance (AR) Highlighting

Shows total outstanding accounts receivable. The Outstanding (AR) KPI card displays the sum of all customer balances, giving instant visibility into total receivables.

Technical Details
  • Calculation: customers.reduce((sum, c) => sum + c.currentBalance, 0)
  • Format: ZAR currency with 2 decimals
  • Real-time: Updates when customer balances change
6

Credit Limit vs Balance Indicator

The Balance (AR) column shows current outstanding balance. When combined with credit limit data (visible in detail view), this helps monitor customer credit utilization.

Technical Details
  • Field: customer.currentBalance
  • Credit Check: Compare with customer.creditLimit
  • Warning: Visual indicator when balance exceeds limit
  • Format: ZAR currency formatting
7

Status Badges

Visual indicators for customer account status. Active (green) means normal operations, Inactive (gray) for dormant accounts, Blocked (red) for suspended customers.

Technical Details
  • Component: Badge with dynamic className
  • Status Values: active, inactive, blocked
  • Colors: Green (active), Gray (inactive), Red (blocked)
  • Function: getStatusColor(status)
8

Actions Menu

Dropdown menu with customer actions: View Details (see full profile), Edit (update customer data), Create Invoice, View Statements, and Delete customer.

Technical Details
  • Component: DropdownMenu with trigger button
  • Actions: View, Edit, Delete
  • Navigation: Opens respective dialogs or pages
  • Permissions: Some actions may be role-restricted
9

Overdue Amount Display

Highlights customers with overdue invoices in orange/red. The Overdue KPI card shows total overdue across all customers, helping prioritize collections.

Technical Details
  • Calculation: customers.reduce((sum, c) => sum + c.overdueAmount, 0)
  • Conditional Styling: Orange text when overdueAmount > 0
  • Source: Calculated from invoice due dates vs current date
10

AR Aging Quick View

Dedicated section showing customers with overdue balances. Each row displays customer name, payment terms, and overdue amount for quick collections prioritization.

Technical Details
  • Filter: customers.filter(c => c.overdueAmount > 0)
  • Display: Orange-themed cards with overdue amounts
  • Purpose: Quick identification of collection priorities
  • Conditional: Only shows if overdue customers exist
