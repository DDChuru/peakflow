Screen Guide

Click on numbered badges or hover over UI elements to learn more about each feature.

1

Create PO Button

Primary action to initiate a new purchase order. Opens a dialog with vendor selection, line items, delivery details, and approval workflow.

Technical Details
  • Action: Opens create dialog
  • Validation: Zod schema with React Hook Form
  • Service: createPurchaseOrderService(companyId, userId)
2

Summary Cards

Real-time statistics showing total POs, pending approval count, approved count, and aggregate value across all purchase orders.

Technical Details
  • Data: Calculated from purchaseOrders state
  • Updates: Real-time on data refresh
  • Formula: .filter().reduce() aggregations
3

Search & Filter Controls

Filter POs by text search (number/vendor), status (draft/pending/approved), and vendor. Real-time filtering with instant results.

Technical Details
  • State: searchTerm, statusFilter, vendorFilter
  • Method: Client-side filtering with .filter()
  • Performance: Debounced search input
4

PO Number

System-generated unique identifier following format PO-YYYY-#### with auto-incrementing sequence number.

Technical Details
  • Format: PO-YYYY-#### (e.g., PO-2025-001)
  • Generation: Server-side sequence counter
  • Uniqueness: Firestore transaction-safe
5

Vendor Name

Linked vendor/supplier from the creditors master file. Click to view full vendor details and transaction history.

Technical Details
  • Source: CreditorService.getCreditors()
  • Relation: po.vendorId → creditor.id
  • Display: po.vendorName
6

Total Amount

Calculated total including line item subtotals plus VAT/tax. Automatically updated when line items or tax rates change.

Technical Details
  • Calculation: subtotal + taxAmount
  • Currency: ZAR (South African Rand)
  • Format: .toLocaleString('en-ZA')
7

Status Badge

Visual workflow indicator: Draft → Pending Approval → Approved → Sent → Received → Closed. Color-coded for quick scanning.

Technical Details
  • Type: PurchaseOrderStatus enum
  • Colors: Gray (draft), Yellow (pending), Green (approved)
  • Function: getStatusColor(status)
8

Actions Menu

Contextual dropdown with status-dependent actions: View, Edit (draft only), Submit for Approval, Approve (admins), Mark as Sent, Cancel.

Technical Details
  • Component: Radix UI DropdownMenu
  • Actions: submitForApproval(), approvePurchaseOrder()
  • Permissions: Role-based (admin/financial_admin)
