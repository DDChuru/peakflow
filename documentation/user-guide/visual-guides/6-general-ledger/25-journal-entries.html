Screen Guide

Click on numbered badges or hover over UI elements to learn more about each feature.

1

Journal Entries Page Header

Main heading identifying this as the journal entries view where all posted transactions are displayed.

2

Create Manual Journal Button

Allows users to create manual journal entries for adjustments, accruals, depreciation, or other GL postings not generated from source documents.

Technical Details
  • Component: Button component with onClick handler
  • Action: Opens manual journal entry form
  • Permissions: Requires financial_admin or admin role
3

Date Range Filter

Filter journal entries by posting date. Supports month, quarter, year, or custom date ranges.

Technical Details
  • Component: Date picker with preset ranges
  • State: Filters journalService.getJournalEntriesWithDetails()
  • Default: Current month
4

Account Filter

Filter entries by specific GL account to see all transactions affecting that account.

Technical Details
  • Component: RadixSelect with account search
  • Data Source: Chart of accounts from industry template
  • Filter Logic: Matches any ledger line with selected account
5

Source Filter

Filter by transaction source: Manual, Invoice, Bill, Payment, Bank, etc. Helps identify entry origin.

Technical Details
  • Options: manual, accounts_receivable, accounts_payable, cash, bank, general
  • Type: JournalEntrySource enum
  • Usage: Audit trail and troubleshooting
6

Journal Entries Table

Main data table showing Date, JE Number, Source, Description, Debit, Credit, Status. Sortable and clickable for details.

Technical Details
  • Data Source: journalService.getJournalEntriesWithDetails(companyId)
  • Returns: Array<JournalEntry & { ledgerEntries: LedgerEntry[] }>
  • Features: Row hover, click to drill down, balanced validation
7

Journal Entry Detail Dialog

Modal dialog showing complete journal entry details including header, ledger lines, dimensions, and audit trail.

Technical Details
  • Component: Dialog from shadcn/ui
  • State: selectedEntry and isDetailDialogOpen
  • Trigger: Click "View" button on table row
8

Ledger Lines in Detail View

Detailed breakdown of all debit and credit lines that make up this journal entry. Shows account code, name, description, and amounts.

Technical Details
  • Source: selectedEntry.ledgerEntries
  • Type: LedgerEntry[] with accountCode, accountName, debit, credit
  • Validation: Total debits must equal total credits
9

Account Code AND Name Display

Phase 1 enhancement - shows both GL account code (e.g., "1200") and full account name (e.g., "Accounts Receivable") for clarity.

Technical Details
  • Fields: line.accountCode and line.accountName
  • Enhancement: Phase 1 - improved readability
  • Display: Code in bold, name in muted text below
10

Line-Level Descriptions

Each ledger line has its own description explaining the specific transaction component (e.g., "Invoice INV-2025-0001", "VAT @ 15%").

Technical Details
  • Field: line.description
  • Usage: Provides context for each posting
  • Display: Shown in muted text, optional field
11

Source Document Link

Shows the origin of this journal entry (Invoice, Payment, Manual, etc.) and provides link to source document when available.

Technical Details
  • Field: entry.source - JournalEntrySource enum
  • Navigation: navigateToSource() function links to invoice/bill page
  • Metadata: Uses entry.metadata.invoiceId for routing
12

Customer/Supplier Dimensions Display

Shows dimensional data (Customer ID, Supplier ID, Invoice ID) associated with the transaction for detailed reporting and analysis.

Technical Details
  • Field: line.dimensions object
  • Common Dimensions: customerId, supplierId, invoiceId, billId
  • Purpose: Enables customer/supplier-level reporting
13

Audit Trail (Created by, Date)

Complete audit information showing who created the entry, when it was created, and whether it was auto-posted or manual.

Technical Details
  • Fields: createdBy, createdAt, updatedBy, updatedAt
  • Auto-posted: Shows "System (Auto-posted)" for invoice/bill entries
  • Compliance: Required for audit and SOX compliance
