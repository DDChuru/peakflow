Screen Guide

Click on numbered badges or hover over UI elements to learn more about each feature.

1

Bank Account Selector

Choose which bank account to import transactions from. Shows masked account numbers for security.

Technical Details
  • Data Source: bankAccountService.listBankAccounts()
  • State: selectedBankAccount
  • Type: Native HTML select
2

Upload PDF Button (with Gemini AI Badge)

Upload bank statement PDFs. Gemini AI extracts transactions, balances, and account details automatically.

Technical Details
  • Component: BankStatementUpload
  • AI: Firebase Functions + Gemini API
  • Processing: OCR and pattern extraction
3

Industry Patterns Info

Shows how many transaction patterns are loaded from your industry template (Technology & Software).

Technical Details
  • Service: IndustryTemplateService
  • Data: Pattern-based GL mapping rules
  • Purpose: Auto-match transactions to accounts
4

Vendor Mappings

Number of known vendors that can be automatically recognized in transaction descriptions.

Technical Details
  • Type: Vendor-specific GL rules
  • Metadata: vendor, category, priority
  • Match: Regex + fuzzy matching
5

Auto-Match Percentage Indicator

Expected auto-mapping success rate based on industry patterns and vendor rules (75-88%).

Technical Details
  • Calculation: Historical auto-match rate
  • Pipeline: MappingPipeline with confidence thresholds
  • Buckets: Auto (≥85%), Review (60-84%), AI (<60%)
6

Tabs (Import / Upload / History / Rules)

Navigate between importing transactions, uploading statements, viewing history, and managing mapping rules.

Technical Details
  • Component: Tabs from @/components/ui/tabs
  • State: activeTab (import | upload | history | rules)
  • Render: Conditional based on tab selection
7

Extracted Transactions Preview Table

Shows transactions extracted from PDF with AI-suggested GL mappings and confidence scores.

Technical Details
  • Data: transactions[] from BankStatement
  • Mapping: MappingPipeline.processTransactions()
  • Confidence: 0-100% based on rule matches
8

Direct Bank-to-Ledger Option

For SMEs: skip reconciliation and post transactions directly to GL after import.

Technical Details
  • Service: BankToLedgerService
  • Action: Creates journal entries immediately
  • Use Case: Small businesses without formal reconciliation
9

GL Account Mapping Dropdowns

Shows suggested debit/credit accounts for each transaction. Users can override if needed.

Technical Details
  • Component: RadixSelect
  • Data: glAccounts[] from COA
  • Format: code - name (type)
10

Save Transactions Button

Posts all mapped transactions to the general ledger. Creates journal entries for each transaction.

Technical Details
  • Action: postToLedger()
  • Validation: All transactions must have GL mappings
  • Result: Journal entries + import session record
11

Import Statistics Panel

Dashboard showing import activity: transactions processed, auto-map rate, total value, active rules.

Technical Details
  • State: statistics object
  • Calculation: calculateStatistics()
  • Metrics: Count, value, percentage, category breakdown
12

BankToLedgerImport Component

Core component providing AI-powered tri-state mapping: Auto-Mapped (≥85%), Needs Review (60-84%), Needs AI (<60%).

Technical Details
  • Component: /src/components/banking/BankToLedgerImport.tsx
  • Pipeline: MappingPipeline + RuleLearningService
  • Features: AI artifact, batch mapping, rule creation
