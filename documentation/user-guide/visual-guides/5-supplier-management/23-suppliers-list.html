Screen Guide

Click on numbered badges or hover over UI elements to learn more about each feature.

1

Add Supplier Button

Primary action to create a new supplier record. Opens a dialog with comprehensive supplier form including basic info, payment terms, bank details, and primary contact.

Technical Details
  • Component: Button with onClick handler
  • Action: Opens create supplier dialog
  • Form: React Hook Form with Zod validation
  • Service: CreditorService.createCreditor()
  • Shortcut: Ctrl/Cmd + N
2

Search Bar

Real-time search across supplier name, email, phone, and category. Supports keyboard shortcut (⌘K) for quick access. Filters table results instantly as you type.

Technical Details
  • Component: Input with search icon
  • State: searchTerm state variable
  • Filter Logic: Client-side filtering using .filter()
  • Search Fields: name, email, phone, category
  • Keyboard Shortcut: Ctrl/Cmd + K
3

Filter by Category

Advanced filtering options for suppliers. Filter by status (active/inactive), creditor type (trade, tax authority, statutory, utility), category, payment terms, or AP balance ranges.

Technical Details
  • Component: Button with Filter icon
  • Filters Available: Status, Type, Category, Payment Terms, Balance
  • Implementation: Client-side filtering with multiple criteria
  • Future Enhancement: Dropdown menu with filter checkboxes
4

Supplier Table

Comprehensive supplier directory displaying: Name, Contact (email/phone), Category, Status, Current AP Balance, and Payment Terms. Click any row to view/edit supplier details. Hover to reveal action menu.

Technical Details
  • Component: Card with scrollable table
  • Data Source: CreditorService.getCreditors()
  • Type: Creditor[] from Firestore
  • Columns: 7 columns with supplier info and actions
  • Interactions: Row click opens edit dialog, hover shows menu
5

Current AP Balance Summary

Displays total accounts payable across all active suppliers. Updates in real-time as bills are recorded or payments made. Critical metric for cash flow management.

Technical Details
  • Component: Summary Card
  • Calculation: suppliers.reduce((sum, s) => sum + s.currentBalance, 0)
  • Data Source: Aggregated from Creditor.currentBalance
  • Updates: Real-time from Firestore listener
6

Payment Terms Display

Shows agreed payment terms in days (e.g., Net 30, Net 7). Used to calculate due dates for bills and track payment compliance. Can include early payment discounts (2/10 Net 30).

Technical Details
  • Field: Creditor.paymentTerms (number of days)
  • Display: "{days} days" format
  • Usage: Auto-calculates bill due dates
  • Common Values: 7, 15, 30, 60, 90 days
7

Creditor Type Badge

Visual indicator of supplier type: Trade (regular suppliers), Tax Authority (SARS), Statutory (UIF, pension funds), Utility (Eskom, municipality), or Other. Helps categorize and prioritize payments.

Technical Details
  • Field: Creditor.creditorType
  • Values: trade, tax-authority, statutory, utility, other
  • Function: getCreditorTypeInfo() returns label and color
  • Component: Color-coded Badge
8

Status Badges

Shows supplier account status: Active (green) for current suppliers, Inactive (gray) for dormant accounts. Only active suppliers appear in bill creation dropdowns.

Technical Details
  • Field: Creditor.status
  • Values: active, inactive, suspended
  • Function: getStatusColor() returns badge classes
  • Component: Badge with conditional styling
9

Early Payment Discount Indicator

Displays available early payment discounts (e.g., "2/10 Net 30" means 2% discount if paid within 10 days). Helps optimize cash management by identifying cost-saving opportunities.

Technical Details
  • Field: Creditor.earlyPaymentDiscount (optional)
  • Format: { percentage, days, terms }
  • Display: Icon or text when discount available
  • Future: Auto-calculate discount savings
10

Actions Menu

Quick actions dropdown: View Details (full supplier profile), Edit (update info), Create PO (new purchase order), Create Bill (record expense), Pay (process payment), Delete (remove supplier).

Technical Details
  • Component: DropdownMenu with DropdownMenuItems
  • Actions: View, Edit, Create PO, Create Bill, Pay, Delete
  • Handlers: openViewDialog(), openEditDialog(), openDeleteDialog()
  • Visibility: Hidden until row hover (opacity-0 group-hover:opacity-100)
